The sf package

The leaflet() function expects latitude and longitude, not eastings and northings. Thereâ€™s a number of ways to make the conversion, but Iâ€™m going to use the sf (â€˜special featuresâ€™) package.

library(sf)

rescue_sf <- rescue %>% 
  sf::st_as_sf(
    coords = c("Easting_rounded", "Northing_rounded"),  # columns with coordinates
    crs = 27700  # coordinate reference system code for eastings/northings
  ) %>% 
  sf::st_transform(crs = 4326)  # the coord ref system code for latlong

print(rescue_sf)

So what happened?

Our eastings and northings have been combined with st_as_sf() into a two element list-column called geometry that has type sfc_POINT. Our object also contains some metadata detailing the geometry type â€“ points in our case â€“ and the projection system of the coordinate data, which we converted to latlong with the st_transform() function.

Functions in the the sf package add an additional class â€“ the â€˜sfâ€™ class â€“ to our dataframe-class object. This makes it possible to perform spatial analysis.

dplyr-friendly

One advantage of using sf is that sf-class objects use the tidy data paradigm that allows for manipulation using elements of the tidyverse. This is a familiar and simple concept to grasp for many people, particularly R beginners. Conversely, the sp package creates the â€˜Spatial Points Data Frameâ€™ (SPDF) class for points data. SPDFs are an S4 class, which means they have â€˜slotsâ€™ of data, coordinates, etc. This is probably slightly less wieldy for beginners.

So for example we can use the filter() function on our sf object.

filtered_sf <- rescue_sf %>%
  dplyr::filter(
    AnimalGroupParent %in% c("Dog", "Cat", "Bird"),  # just these animals
    DateTimeOfCall > lubridate::ymd("2017-01-01") &
      DateTimeOfCall < lubridate::ymd("2017-12-31")  # 2017 only
  )

Map it

So now we can plot the data. The addMarkers() function in leaflet will accept our geometry list-column and plot the latitude and longitude of each point.

This is a very simple map for now. You can:

  • zoom with the + and - buttons or scroll with your mouse wheel
  • click and drag to move the map
  • click the points to get a pop-up showing more information
library(leaflet)

filtered_sf %>% 
  leaflet::leaflet() %>% 
  leaflet::addTiles() %>% 
  leaflet::addAwesomeMarkers(
    icon = leaflet::awesomeIcons(
      library = "ion",
      icon = "ion-android-alert",
      iconColor = "white",
      # colour by animal
      markerColor = case_when(
        filtered_sf$AnimalGroupParent == "Dog" ~ "red",
        filtered_sf$AnimalGroupParent == "Cat" ~ "blue",
        filtered_sf$AnimalGroupParent == "Bird" ~ "black"
      )
    ),
    popup = ~paste0(
      "<b>Animal</b>: ", filtered_sf$AnimalGroupParent, "<br>",
      "<b>Incident</b>: ", filtered_sf$FinalDescription, "<br>",
      "<b>Date</b>: ", filtered_sf$DateTimeOfCall, "<br>",
      "<b>Borough</b>: ", filtered_sf$Borough, "<br>",
      "<b>Nominal cost</b>: Â£", filtered_sf$IncidentNominalCost
    )
  )
