Three things

So I spoke about three very broad things to achieve reproducibility. I focused on R and some specific packages that could be helpful, but the ideas are transferable.

1. Centralise everything

Get code, functions, data, documentation in one place. Use R Projects in RStudio and write packages.

2. Report with code

Put code inside your report so that updates to data and code will be reflected instantly. Use R Markdown and other formats like {xaringan} for reproducible slides and {bookdown} for reproducible books.

3. Manage workflows

Don’t use your brain to store information about the dependencies within your analysis. Use {drake} instead. It remembers all the relationships between the files, objects and fcuntions in your analysis and only re-runs what needs to be re-run following changes.
