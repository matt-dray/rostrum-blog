tl;dr

The {r.oguelike} package—a toy roguelike game for the R console—now has sound effects thanks to {sonify}. Pew pew!

Hi-Sonifi

Anyway, yes: {r.oguelike} now has sound effects with quality as high as its graphics and gameplay.

I used the {sonify} package to create little beeps and toots that I think fit the game’s aesthetic. These are fired as the player moves and interacts with things in the dungeon.

I’ve written about {sonify} before when I sonified data about COVID-19 infections and GitHub activity (incredible juxtaposition).

Basically you can make an audible plot, which offers a more interesting and accessible way of presenting data. A bell curve will sound like woooOOOOooop, for example.

You can also hijack {sonify} to make funny little honks and parps, which is what I’ve done for {r.oguelike}. I wrote about this for {ActionSquirrel}, a sort-of proto-{r.oguelike}.

How did I arrive at the sounds for {r.oguelike}? I did the bare minimum of fiddling around with the arguments in sonify::sonify() until the noises amused me.

Demo cassette

The outputs are {tuneR} objects, which I’ve saved as wav files with tuneR::writeWav() so they can be embedded below.

In reality, the sounds play a little slower in the game, but it was a bit fiddly to reproduce it for these clips. You get the idea.

Click for illustrative code to create the wav files. 
library(sonify)
library(tuneR)
library(purrr)

sfx <- list(
  
  move = sonify(1, 1, duration = 0.001),
  
  bump = sonify(1, 1, duration = 0.01, flim = c(100, 110)),
  
  gold = bind(
    sonify(1, 1, duration = 0.05, flim = c(800, 800)),
    sonify(1, 1, duration = 0.05, flim = c(1000, 1000))
  ),
  
  apple = sonify(0:1, c(0, 1), duration = 0.05),
  
  eat = sonify(0:1, c(1, 0), duration = 0.05),
  
  win = bind(
    sonify(0:1, rep(1, 2), duration = 0.1, flim = c(600, 600)),
    sonify(0:1, rep(1, 2), duration = 0.1, flim = c(600, 600)),
    sonify(0:1, rep(1, 2), duration = 0.1, flim = c(800, 800))
  ),
  
  lose = bind(
    sonify(0:1, rep(1, 2), duration = 0.1, flim = c(600, 600)),
    sonify(0:1, rep(1, 2), duration = 0.1, flim = c(600, 600)),
    sonify(0:1, rep(1, 2), duration = 0.1, flim = c(400, 400))
  )
  
)

walk2(
  .x = sfx,
  .y = names(sfx), 
  ~writeWave(.x, paste0(.y, ".wav"))
)

Move

If you move onto an unoccupied floor tile (.), you get a very quick ‘tap’ like a footstep.

sonify(1, 1, duration = 0.001)

Bump

Bump into the dungeon wall (#) and you’ll get a dull thud, you absolute clod.

sonify(1, 1, duration = 0.01, flim = c(100, 110))

Yes, flim, as in: ‘this post is absolute flimflam’.

Collect gold ($) and get a celebratory ding.
